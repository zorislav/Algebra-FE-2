Uvod u Frontend Development

Frontend development fokusira se na sve što korisnik vidi i s čime izravno komunicira na web stranici.

Uloga HTML-a u web razvoju

HTML je temelj svake web stranice. On definira strukturu sadržaja, kao što su naslovi, paragrafi, slike, liste i ostali elementi.

Osnovna struktura HTML dokumenta

Svaki validan HTML5 dokument ima doctype element na početku. Nakon njega se nalazi html element u kojem se nalaze elementi head i body. Head element uglavnom sadrži elemente koje korisnik ne vidi na samoj web stranici. Body sadrži elemente pomoću kojih se definira i strukturira sadržaj kojeg korisnik vidi.

Naslovi i njihova važnost

Naslovi pomažu čitačima i tražilicama da razumiju strukturu stranice. <h1> se obično koristi samo jednom i predstavlja glavni naslov stranice, dok <h2> i ostali razrađuju podteme.

CSS i vizualni identitet stranice

CSS služi za stiliziranje HTML elemenata. Omogućuje kontrolu nad bojama, fontovima, razmakom, rasporedom elemenata i mnogim drugim vizualnim aspektima.

Zašto je važna konzistentnost dizajna

Konzistentan dizajn korisniku daje osjećaj jasnoće i profesionalnosti. Korištenje pravilne tipografije, razmaka i boja doprinosi boljoj čitljivosti i usmjeravanju pažnje na bitne dijelove sadržaja.

Primjer dobre prakse

Preporučuje se korištenje relativnih jedinica poput rem kako bi dizajn bio prilagodljiv različitim uređajima i postavkama preglednika.

JavaScript i interaktivnost

JavaScript općenito omogućuje dinamiku na web stranicama. Omogućuje: validaciju obrazaca, animacije, rad s API-jima te puno više. Danas se najčešće koristi uz moderne razvojne okvire kao što su React, Angular ili Vue.

Reaktivni pristup

Reaktivno programiranje omogućuje da se promjene u podatkovnim tokovima (eng. streams) odmah odraze u korisničkom sučelju. Ovo posebno dolazi do izražaja u Angular aplikacijama.

Primjena u praksi

Pomoću RxJS biblioteke moguće je elegantno upravljati asinkronim događajima, API pozivima i promjenama u stanju aplikacije. Primjer je promjena stranice u tablici. Promjena stranice rezultira novim zahtjevom na poslužitelj za dohvat podataka na toj stranici. Poslužitelj vraća odgovor koji se u JavaScript kodu prezentira u obliku Observable podatkovnog toka. Promjena u podatkovnog toku uzrokuje ažuriranje podataka u tablici koju korisnik vidi.
